# Taskflow

Automated task triage and implementation from project management tools like ClickUp.

Taskflow is a Claude Code plugin that pulls tasks from your project management tool, classifies them, creates implementation plans, groups them into smart batches, and then implements each batch — complete with worktrees, tests, and PRs.

## Installation

```bash
claude plugins marketplace add github:rentorious/taskflow
claude plugins install taskflow@taskflow
```

After installing, restart Claude Code for the skills to become available.

## Quick Start

1. **Setup** — Run `/taskflow:setup` to generate `.claude/taskflow-config.json` for your project. The wizard auto-detects your codebase structure and walks you through connecting your project management tool.

2. **Triage** — Run `/taskflow:triage` to pull your assigned to-do tasks, classify them, create per-task implementation plans, and group them into batches.

3. **Implement** — Run `/taskflow:implement` to execute the next batch: creates a worktree, implements changes, runs checks, updates task statuses, and opens a PR.

## Skills

| Skill | Command | Purpose |
|-------|---------|---------|
| Setup | `/taskflow:setup` | Interactive wizard to generate project config |
| Triage | `/taskflow:triage` | Fetch, classify, plan, and batch tasks |
| Implement | `/taskflow:implement` | Execute a batch: implement, verify, PR |

## Supported Providers

- **ClickUp** — fully supported
- **Jira** — planned

## Config Reference

The config file lives at `.claude/taskflow-config.json` in your project. It's generated by `/taskflow:setup` and consumed by the other skills.

| Field | Description |
|-------|-------------|
| `version` | Schema version (currently `1`) |
| `provider` | Project management tool (`"clickup"`) |
| `project_name` | Used for worktree naming |
| `clickup` | Provider-specific config (workspace, lists) |
| `areas` | Classification areas with paths and verification commands |
| `extra_areas` | Non-code classification areas (e.g., `"sendgrid"`, `"manual"`) |
| `branch_conventions` | Branch prefix mapping by task type |
| `output_dir` | Where state/plan files are written |
| `base_branch` | Branch to create worktrees from |
| `build_before_implement` | Commands to run in worktree before implementation |
| `install_command` | Dependency installation command |
| `full_lint` | Full-repo lint command |
| `full_typecheck` | Full-repo typecheck command |
